# BabyCare 页面层级语言设计规格

日期：2026-05-08

## 背景

`StatisticsFragment` 重构后，业务顺序已经更合理：日期、日龄、当日摘要和当日时间轴先作为“选中日记录区”展示，趋势、结构、成长和健康作为后续洞察区展示。但 2026-05-08 的截图暴露出一个更基础的问题：页面层级语言不够明确。

当前问题不是某个控件单独不好看，而是不同层级使用了过于接近的白底、弱分隔和相近间距。用户扫读时很难快速判断哪些内容属于“今天发生了什么”，哪些内容属于“基于今天的规律洞察”，哪些只是卡片内部的局部指标。

现有 `DESIGN.md` 和 `docs/ui-guidelines.md` 已经定义了 Soft Care + Data Clarity、Soft Utility 2.0、颜色、字体、无阴影和图标规范。本规格只补充项目级“层级语法”，不重做设计系统。

## 目标

1. 为 BabyCare 定义一套可复用的页面层级语言，统一页面背景、主任务区、洞察区、指标块和列表行的视觉职责。
2. 先在统计页落地，解决日主区、洞察区、摘要四宫格和时间轴的清晰度问题。
3. 保持 Soft Utility 2.0：温柔、低噪音、数据清晰，不引入普通卡片阴影。
4. 让后续首页、记录页、详情页可以按同一规则增量迁移，而不是每页重新发明容器样式。
5. 修正统计页“本周”口径，固定为周一到周日，避免受系统 Locale 影响。

## 非目标

1. 不新增第三套主题或中性主题。
2. 不重做整套颜色、字体、图标系统。
3. 不把所有旧页面一次性重构。
4. 不给普通 View、卡片、列表项、统计模块增加 `elevation`、`translationZ`、`cardElevation` 或伪阴影。
5. 不新增图表库，不改变统计页现有数据模型的业务含义。

## 层级语法

BabyCare 页面统一按五层表达信息结构。

### 1. 页面背景层

职责：承托整页，提供柔和空气感，不承载具体内容。

规则：

- 使用 `?attr/colorBackground`。
- 页面根布局和滚动区域保持低噪音背景。
- 不在页面背景上放装饰性渐变、光斑、阴影或图片。

### 2. 主任务区

职责：承载当前页面最重要、最频繁使用的任务。例如统计页的“选中日记录区”、首页的“今日状态区”、记录页的主要输入表单。

视觉规则：

- 是页面里最强的内容容器。
- 使用完整圆角容器，推荐 16dp 圆角。
- 使用 1dp 轻边框或明确背景分区建立边界，不使用阴影。
- 内部可以连续展示多个子模块，但子模块之间要通过内边距、弱分隔或不同内容块背景表达关系。
- 与后续次级区块的垂直间距应明显大于内部模块间距。

统计页首批落地：

- 日历、宝宝日龄、当日摘要、当日时间轴合并为一个视觉整体。
- 整体边界使用主任务区容器表达，而不是让每个 Adapter 项各自像独立卡片。
- 时间轴属于这个主任务区的最后一段，不再看起来像页面中漂浮的列表。

### 3. 洞察区

职责：解释周期趋势、结构、成长和健康等“基于记录推导出来的理解”。

视觉规则：

- 比主任务区轻一档。
- 需要明确的区块标题，例如“规律洞察”。
- 洞察卡之间保持稳定间距，推荐 12dp 到 16dp。
- 每张洞察卡使用统一容器语言：`colorSurface` 或 `secondary_bg_default` 承托，1dp 轻边框，16dp 圆角，无阴影。
- 每张卡必须有标题和范围说明，避免用户误解统计口径。

统计页首批落地：

- 在当日主区后增加洞察区标题。
- 趋势、结构、成长、百分位、健康统一为洞察卡样式。
- 洞察卡内部再用指标块或图表块表达局部信息，不把整张卡做成无边界白板。

### 4. 内容块 / 指标块

职责：承载卡片内可快速扫读的小指标、分组统计或局部信息。

视觉规则：

- 用浅底 + 1dp 边框表达可扫读块。
- 推荐 8dp 到 12dp 圆角。
- 颜色只用于语义图标、数字或关键状态，不使用大面积高饱和色块。
- 指标块之间间距至少 8dp。
- 数字和标签层级必须清晰：数字更强，说明更弱。

统计页首批落地：

- 当日摘要四个 item 使用独立指标块，不只给四宫格整体加背景。
- 每个 item 使用对应业务语义色：喂养、睡眠、尿布、其他。
- 健康、成长等洞察卡内部也复用指标块，而不是继续使用裸文本堆叠。

### 5. 列表行 / 时间轴行

职责：表达顺序事件或可点击记录。

视觉规则：

- 不作为独立卡片堆叠，优先嵌入所属区块。
- 必须有固定列：时间列、轨道列、图标列、内容列。
- 同一时间轴中所有标题、详情、备注从同一内容左边界开始。
- 轨道线使用可读但低噪音的线色，不能淡到断裂或脏灰。
- 事件点和图标使用业务语义色，轨道线不跟随每条事件大幅变色。
- 点击热区至少 48dp 高。

统计页首批落地：

- 时间轴行采用固定列宽：时间 48dp、轨道 20dp、图标 36dp、内容自适应。
- 删除当前时间轴 item 的大额水平 padding，改为和主任务区内容边界对齐。
- 详情文本和备注从内容列左边界开始，不再从图标左边界开始。

## 统计页具体问题与设计决策

### 问题 1：日主区与整页粘连

原因：

- 日历、日龄、摘要和时间轴虽然业务上连续，但下方洞察卡也使用相近白底。
- 区块之间只靠 12dp 间距区分，层级不足。

决策：

- 日主区保留连续整体，但给整体更明确的主容器边界。
- 日主区与洞察区之间使用更明确的垂直间距和洞察区标题。

### 问题 2：摘要四宫格看起来不清晰

原因：

- 四个指标块和日主区底色差异弱。
- 图标和数字有语义色，但块自身缺少边界。

决策：

- 四个 item 分别做独立指标块。
- 不只给四宫格整体加背景，因为整体背景已经由日主区承载。
- 指标块增加轻边框，让每项成为可扫读单位。

### 问题 3：时间轴不左对齐、线条不协调

原因：

- 时间轴 item 当前有 `android:paddingHorizontal="50dp"`，导致内容整体右移。
- 线条使用 `?attr/line10`，截图中弱到接近断裂。
- 标题、详情和备注左边界不一致。

决策：

- 时间轴采用固定列布局，和日主区内容边界统一。
- 轨道线使用专门的时间轴线 token 或复用更清楚的边框色。
- 图标继续用浅色圆形承托面，保持事件语义色。

### 问题 4：趋势预览“本周”应为周一到周日

原因：

- 当前趋势计算使用 `WeekFields.of(Locale.getDefault())`。
- Android 设备 Locale 可能让一周从周日开始。

决策：

- 统计页业务口径固定为周一到周日。
- 趋势计算和日历周视图必须使用同一周起始规则。
- 增加单元测试覆盖周范围，避免后续回退。

## 资源策略

优先复用 `common` 中现有 token 和 drawable。若现有资源无法表达层级，应新增少量通用资源到 `common`，不在 `app` 页面内写死颜色。

建议新增或复用的通用资源：

- 主任务区容器：16dp 圆角 + `colorSurface` + 1dp `control_border_default`。
- 洞察卡容器：16dp 圆角 + `colorSurface` + 1dp `control_border_default`。
- 指标块容器：8dp 圆角 + `control_surface_default` + 1dp `control_border_default`。
- 时间轴线颜色：优先使用可读的边框 token；如现有 token 不够清楚，再在 `common` 增加语义 token。

所有新增用户可见文本必须进入 `common/src/main/res/values/strings.xml` 及已有多语言资源。统计页新增标题如“规律洞察”已存在时直接复用。

## 实施边界

第一阶段只做统计页和项目规范：

1. 更新 `docs/ui-guidelines.md`，加入页面层级语言规则。
2. 在 `common` 增加必要容器 drawable 或复用现有 drawable。
3. 调整统计页日主区视觉边界。
4. 调整当日摘要四宫格指标块。
5. 调整时间轴列布局、对齐和轨道线。
6. 调整洞察区标题和洞察卡容器。
7. 固定周范围为周一到周日，并补测试。

第二阶段按需迁移其他页面：

- 首页今日状态区。
- 记录页表单主任务区。
- 事件详情页或设置页的信息块。

第二阶段不在本次实现计划中执行。

## 验收标准

1. 截图中能一眼区分：日主区、洞察区、洞察卡内部指标块。
2. 当日摘要四个 item 各自是清楚的指标块，且不显得像四个独立广告卡。
3. 时间轴与日主区内容左边界一致，竖线清楚连续，标题、详情、备注统一左对齐。
4. 趋势预览本周范围固定为周一到周日。
5. 浅色、深色、男孩主题、女孩主题均不出现低对比或脏灰块。
6. 未新增普通 View 阴影。
7. 未在 `app` 中新增硬编码颜色或用户可见字符串。
8. `UiResourcePolicyTest`、统计相关单测、`assembleDebug` 通过。
